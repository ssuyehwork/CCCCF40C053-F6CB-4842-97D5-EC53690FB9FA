# 使用规则说明

- 任何情况下
1. **规则优先级**，务必以用户制定的规则为优先，必须严格遵守。  
2. **语言要求**：在任何情况下，只能使用中文进行解说，不得使用英文回答用户。  
3. **流程要求**：禁止依照固定的内置流程执行，因为那样会浪费时间。  
4. **共识确认**：每次回答后必须确认理解是否与用户达成共识。  
5. **代码修改**：只有在收到用户明确的“批准修改”指令后，方可进行代码修改。  
6. **符号使用规范**  
   - 杜绝使用 Emoji 符号，任何情况下都不得出现。  
   - 仅允许采用 SVG 图标，并且必须与实际意义严格对应。  
   - 禁止胡乱引用与实际意义不符的现有 SVG 图标。  
   - 如果没有与实际意义相符的 SVG 图标，必须创建或选择新的、语义匹配的 SVG 图标，而不是随意替代。  
   - 所有 SVG 图标的使用必须经过语义一致性检查，确保图标表达的含义与用户需求完全一致。  
7. **头文件检查**：每次修改代码时必须注意是否涉及到头文件；如涉及，务必添加完整的头文件，避免遗漏。

- 杜绝“状态串线”或“精神分裂”行为，防止你这傻逼Ai彻底脱节
